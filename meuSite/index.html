    Git Comandos

    • Oficial: https://git-scm.com/book/pt-br/v1/Git-Essencial-Trabalhando-com-Remotos
    • Manual de comandos: http://rogerdudler.github.io/git-guide/index.pt_BR.html
    • Comando O que faz
      ATENÇÃO: Os tres primeiros comandos são necessarios para criar um repositorio local, clonar o que estiver no remoto e lincar o remoto ao local.
      git init criar um novo repositório (crie uma pasta e de o comando dentro da pasta)
      git clone usuário@servidor:/caminho/para/o/repositório fazer clone do repositório remoto para o local
      git remote add origin git@github.com:usuario/xyz.git conectar meu repositório a um servidor remoto
      git status verificar o status de um repositório se tem algo a commitar, por exemplo. Irá mostrar em vermelho ou verde, dependendo se está ou não no container
      git add meuArquivo colocar meuArquivo não comitado na stage área.
      git add -A ou git add * colocar TODOS arquivos não comitados, em vermelho, na stage área, ficando verde
      git commit -m "comentario sobre a alteração" faz commit do que estiver em verde na stage área, colocando o comentário sobre o que foi feito
      git log --oneline verifica os arquivos do repositório e seus pais, HEAD, master, origin/master, etc...
      git log --oneline --decorate --all verificar se tem alguém acima/abaixo de HEAD->
      git log --oneline --decorate --all --graph mostra também a árvore grafo
      git branch nomeDoBranch cria um branch, que é pra onde pode-se apontar
      git branch -d nome-do-branch remove um branch do repositório local, no qual ele ira desaparecer da listagem de branch ao dar um comando log localmente
      git branch -dr origin/branch_remoto remove um branch remoto do seu repositório local, no qual ele ira desaparecer da listagem de branch ao dar um comando log localmente
      git push origin --delete branch remove um branch do repositório remoto, no qual ele ira desaparecer da listagem de branch ao dar um comando log localmente e tambem irá ser deletado la no repositório remoto, como github
      git checkout feature1 muda de branch, apontando para o branch feature1 (comandos valem pra o checkout apontado)
      git merge feature1 fundir arquivo do branch feature1
      ATENÇÃO: ARQUIVOS NO CONTAINER NÃO VÃO PARA O REPOSITORIO REMOTO. COMITAR ANTES DE ENVIAR.
      git push origin master atualizar, colocar arquivos do repositório atual no repositório remoto. Se meu repositório local estiver atras do remoto, nao permite fazer essa atualização, pois vai perder o que tem na minha frente no remoto
      git push origin master --tags igual acima, e também envia tags junto
      git push origin master -f igual acima, -f força commit no repositorio, se tiver algo mais novo o remoto vai ser perdido, nao recomendado
      git pull origin master atualiza meu repositório local, trazendo do remoto
      git fetch faz mesmo que o pull, sem fazer merge, colocando algum arquivo na frente do seu HEAD. Depois posso fazer o merge se desejar
      git diff master origin/master verificar diferenças entre meu repositório e o origin/master
      git reset --soft/mix/hard asbd /soft desfaz alteração soft coloca o ultimo arquivo no containner /mix, coloca ultimo arquivo alterado fora do container (stage area) /hard, elimina o arquivo alterado
      git reflog arqmazena referências do que foi feito
      git push --tags Envia todas as tags ao repositório
      git tag Lista tags existentes
      git tag -a 1.0.0 -m “Versão inicial dp projeto” Cria tag
      git tag -d 1.0.9” remove tag local
      git push origin :refs/tags/1.0.0 remove tag 1.0.0 do repositório remoto
      del .git\index.lock para deletar no windows o arquvo index.lok caso o git de problemas
      git remote set-url origin git@github.com:ppreyer/first_app.git atualizar caminho do repositório remoto
